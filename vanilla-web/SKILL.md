# vanilla-web тАФ how websites get built here

**No build step, no runtime deps** тАФ plain ES modules served statically; the
only dev dependency is `typescript`. The gate is two halves:
`tsc --noEmit` for the JS and `check-css-vars` for `var(--x)` (тЖТ `reference/modules.md`).

## Decision rule

New web UI тЖТ default to this stack. Reach for React (or another framework) only
with a concrete driver: heavy shared client state across many screens, an
existing React codebase, or a component ecosystem the task genuinely needs (rich
editors, complex drag-drop). "It might grow" is not a driver. If React seems
warranted, say so and ask before scaffolding.

## App skeleton тАФ always multi-view

Every app starts as views + registry, even with one view (a single page is a
multi-view app with one entry; the skeleton is then already there when page two
arrives).

```

тФВ                        #   <main id="stage">, modulepreload + module script
```

Copy **verbatim** from the skill dir: `shell.js` (makes `location.hash`
`#/<view-id>` the source of truth тАФ deep links + back button free; one
`AbortController` per mount; `document.startViewTransition` swaps; surfaces
errors to `<output id="errbar">`), `lib/templates.js`, `serve.mjs`,
`tsconfig.json`, `tools/check-css-vars.mjs` тАФ plus `preview.*` + `previews/scan.mjs` if you want the
component catalogue (тЖТ `reference/preview.md`). `index.html` preloads the module graph
(`modulepreload` for `shell.js`, `views/registry.js`, `lib/templates.js`).

## The view contract

Everything a view starts goes through `helpers.signal`, so teardown is
structural, not a checklist:

```js
let cssLink;
export default {
  id: "overview",
  async mount(container, data, { loadCSS, every, signal }) {
    cssLink = loadCSS(import.meta.url, "./style.css");
    await loadTemplates(new URL("./overview.html", import.meta.url).href);
    btn.addEventListener("click", onClick, { signal });   // auto-removed
    every(refresh, 5000, signal);                          // auto-cleared
    const es = new EventSource("/api/events");             // see reference/server.md
    signal.addEventListener("abort", () => es.close(), { once: true });
  },
  unmount() { cssLink.remove(); }, // shell aborts the signal before calling this
};
```

## Invariants тАФ these always hold (detail behind each link)

- **No HTML strings in JS.** Markup is `<template id="tpl-тАж">` in `.html` files
  with `data-slot` markers, cloned via `tpl()`/`pick()`. тЖТ `reference/components.md`
- **Components** are `components/<name>/` folders with the factory contract
  `create<Name>(props, signal) тЖТ { el, тАжupdaters }`. тЖТ `reference/components.md`
- **CSS** is `@scope` per component + tokens in `@layer`; responsiveness via
  `@container`, not viewport media. тЖТ `reference/css.md`
- **Re-renders** of polled data go through `renderRegion` (never raw
  `replaceChildren`/`innerHTML`); mutate in place for fast-ticking values.
  тЖТ `reference/interactivity.md`
- **Overlays** use native `<dialog>` / `popover` / `<details>` тАФ never
  hand-rolled. тЖТ `reference/interactivity.md`
- **Forms** use native validation (`required`/`pattern`, `reportValidity()`,
  `:user-invalid`). тЖТ `reference/interactivity.md`
- **Declarative over imperative.** Reach for a platform attribute before a JS
  listener тАФ `command`/`commandfor` and `popovertarget` for overlays,
  `required`/`pattern` for forms тАФ so an element's behaviour reads off its markup;
  `addEventListener` (always `{ signal }`-scoped) is the fallback for what no
  attribute covers. тЖТ `reference/interactivity.md`
- **Pending state** is attribute-driven: a fetch marks its region busy
  (`withPending` тЖТ `aria-busy`/`data-pending`) and CSS renders the busy look.
  тЖТ `reference/interactivity.md`
- **Numbers / dates / durations** render through `Intl` (via `lib/format.js`).
  тЖТ `reference/modules.md`
- **The gate**: every module starts `// @ts-check` + JSDoc; `tsc --noEmit` **and**
  `check-css-vars` (undefined `var(--x)` fails silently). тЖТ `reference/modules.md`
- **Preview** (optional): a component can ship `<name>.preview.js` exporting
  `{ title, render, variants }`; `serve.mjs` generates the catalogue and serves
  it at `/preview.html`. No npm, no build. тЖТ `reference/preview.md`
